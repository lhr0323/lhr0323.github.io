我为什么要考研

发表于 2017-03-25   |   分类于 生活   |  

我一直很想写一篇关于考研的博客，为什么考研有很多人其实并不清楚自己为什么要考只是看见别人都去了自己也想试一试。作为一个非重点学校的大三的学生来说自己即将面对很多的抉择，面临着找工作和考研的双重选择，对任何人来说这都是人生重大的抉择，以后的人生也会随之而改变，作为一个计算机和数学专业的学生考不考研一直都是逃不过的话题但对我来说这并不是一个很难做的判断题，大学这三年对我影响最深的就是编程，对编程一开始的了解只是通过电视，网络，觉得计算机很神奇，觉得很酷，学这个也是因为可以玩电脑开始的，当我第一次接触编程的时候就觉得这是我喜欢的的东西，我热爱它，从最初的C语言到后来的java，C++，php,python,这些语言让我对计算机有了全新的认识和定义，那是觉得自己很厉害好像什么都会，对于平常的课也不去，找个没人的地方去学自己的东西，但学着学着就出现问题了，自己写的东西好像并没有什么用处，自己写的为什么没有别人好，自己到底要向那个方向发展这些在困惑着我 ，有时候觉得自己技术很厉害了但实际上差的很远，也有可能是我的专业带给我的影响我的专业名称信息与计算科学专业俗称计算数学，也是这个专业带给我影响让我去了解本质上的东西，对于计算机什么是本质这是我一直不了解的，一个程序是怎样执行的中间经过了哪些过程这是值得每一个程序员去思考的。计算机是怎样运行的这些东西都是未知的，作为对我影响最深的app应该是知乎，在这上面我找到了一些回答，一些程序员的思考，在上面有一本推荐的书给了我巨大的影响，《深入理解计算机系统》著名的CSAPP，在这本书中以一个程序员的角度来看计算机让我有了全新的认识也知道了普通程序员和大牛之间的差距，但作为一个非重点大学的学生本科阶段的时间并不足以我去对计算机有更深入的理解，自己的功底并不深厚，这也是我考研的原因之一，我希望自己在计算机方面走得更远一些，能走得更远一些的前提就是自己的基础有多扎实，对于未来我觉得机器学习人工智能将会是下一次科技浪潮的开始，在自己的学习中也对这些产生了兴趣，所以我选择考研。

数字游戏

发表于 2016-12-25   |   分类于 竞赛   |  

问题描述
　　栋栋正在和同学们玩一个数字游戏。

　　游戏的规则是这样的：栋栋和同学们一共n个人围坐在一圈。栋栋首先说出数字1。接下来，坐在栋栋左手边的同学要说下一个数字2。再下面的一个同学要从上一个同学说的数字往下数两个数说出来，也就是说4。下一个同学要往下数三个数，说7。依次类推。

　　为了使数字不至于太大，栋栋和同学们约定，当在心中数到 k-1 时，下一个数字从0开始数。例如，当k=13时，栋栋和同学们报出的前几个数依次为：
　　1, 2, 4, 7, 11, 3, 9, 3, 11, 7。

　　游戏进行了一会儿，栋栋想知道，到目前为止，他所有说出的数字的总和是多少。
输入格式
　　输入的第一行包含三个整数 n,k,T，其中 n 和 k 的意义如上面所述，T 表示到目前为止栋栋一共说出的数字个数。
输出格式
　　输出一行，包含一个整数，表示栋栋说出所有数的和。
样例输入
3 13 3
样例输出
17
样例说明
　　栋栋说出的数依次为1, 7, 9，和为17。
数据规模和约定
　　1 < n,k,T < 1,000,000；
*/

分析 每个人说的数字最后是有规律的将输出的规律建一个数组然后从里面寻找到某人说的数字。
注意 K为奇数和偶数时建表方式不同当K为奇数时表为K当K为偶数时表为2K这是出错最多的地方一定要注意。

Median of Two Sorted Arrays

发表于 2016-12-20   |   分类于 算法   |  

Median of Two Sorted Arrays
There are two sorted arrays nums1 and nums2 of size m and n respectively.
Find the median of the two sorted arrays. The overall run time complexity should be O(log (m+n)).

比较重要的题要求时间复杂度为O(log(m+n))遍历的方法只能做到O(M+N)所以是不行的 考虑转换求解方式用分治法或者二分法 变成寻找第k小数的问题 通过递归的方法寻找比较两个数组的中位数大小将大的数组的右半部分扔掉将小的数组的左半部分扔掉 递归到最后得到最终的中位数
